1) что такое обобщенный тип?
- Если элемент языка (класс, структура, интерфейс, метод, делегат) оперирует параметризованным 
типом данных (обобщением), то этот элемент называется обобщенным (обобщенный класс, 
обобщенный метод и т. д.).

2) что такое where?
- это ограничение которое накладывается на параметр обобщенного типа. 
Синтаксис, class myClass<T> where T : тип_ограничения
Варианты типа ограничения:
1. class - параметр должен относится к ссылочному типу
2. struct - параметр должен относится к типу значения
3. myAnotherClass - параметр должен относится к типу класса myAnotherClass
или быть производным от него
4. IControl - параметр должен реализовывать все методы интерфейса IControl
5. new() - тип должен содержать общедоступный конструктор по умолчанию

какой порядок ограничений?
- сначала class, struct, потом IConctrol, myClass, последнее new()

что такое статические обобщенные поля?
- это такие поля обобщенного типа, которые можно проинициализировать в main без создания 
экземпляра этого типа. Например,
public class StatString<T> {public static String name;}
static void Main() {StatString<int>.name="str"; StatString<bool>.name=true}
- При типизации обобщенного класса определенным типом будет создаваться свой набор статических 
членов
- статические поля типа размещается в самом объекте-типе без создания экземпляра этого типа

что такое default в обобщении?
- Он нам нужен для того чтобы задать какому-то полю значение по умолчанию. Если оно будет
ссылочного типа, то выведет 0, тип значений - null
